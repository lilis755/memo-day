# Skill: memo-day-workday-capture

## Use when

当需要把一天中的工作事项、会议、临时任务和后续备注整理成一个可回顾的本地时间线时使用。

## Purpose

把零散待办转成按日期和时间段组织的个人工作记录，减少“今天做了什么、某个时间段发生了什么、周报要回看什么”的回忆成本。

## Inputs

1. 事项标题
2. 日期
3. 开始时间
4. 持续时长
5. 事项颜色分类
6. 时间段备注

## Process

1. 先按日期选择当天视图，不直接堆成普通待办列表。
2. 在时间轴上选择或拖动一个时间段，形成明确的时间边界。
3. 为事项填写标题、开始时间、持续时长和颜色分类。
4. 完成后在日程卡片上补充备注，记录产出、沟通结论或后续线索。
5. 每天结束后通过日历回看当天时间分布，辅助写周报和复盘。

## Output

1. 按天组织的事件列表
2. 可视化时间轴
3. 每个时间段的备注记录
4. 本地持久化的 `events.json`
5. 可用于周报或复盘的工作轨迹

## Constraints

1. 第一版不做账号系统。
2. 第一版不做团队协作。
3. 第一版不做复杂项目管理字段。
4. 数据优先保存在本机，避免把个人工作过程强依赖云端。
5. 事项必须落到具体日期和时间段，不能只停留在泛泛待办。

## Verification

1. 能新增一个事项并在时间轴上显示。
2. 能切换日期并查看历史记录。
3. 能删除事项。
4. 能打开事项备注并保存。
5. 刷新页面后数据仍存在。
6. 本地服务可将数据写入 `events.json`。
